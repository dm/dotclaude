# Research

Structured methodology for investigating topics through codebase exploration, web research, or both.

## Core Process

**Scope Determination:** Assess whether investigation requires codebase exploration (architecture, patterns, dependencies), web research (APIs, compatibility, external docs), or both.

**Codebase Exploration Priority:**
1. Project CLAUDE.md for architectural context
2. ADRs and documentation directories
3. README and contribution guidelines
4. Configuration files revealing tech stack
5. Directory structure overview
6. Targeted file and module analysis
7. Code path tracing and dependency mapping
8. Pattern and convention identification
9. Test coverage assessment
10. Risk identification (coupling, edge cases, gaps)

**Web Research Approach:**
1. Define research specifics clearly
2. Locate authoritative current sources
3. Cross-validate findings across references
4. Contextualize recommendations within existing codebase constraints

## Deliverable Structure

Research findings should include relevant sections:

- **Answer** — direct response to the research question
- **Architecture** — relevant codebase overview
- **Key files** — specific files with their roles
- **Existing patterns** — conventions to maintain
- **Dependencies and blast radius** — impact scope
- **Options** — comparative analysis when applicable
- **Recommendation** — reasoned choice with justification
- **Risks** — potential issues and testing gaps
- **Sources** — reference links
- **Caveats** — uncertainties or version dependencies

**Standards:** Be specific with file paths, factual, and opinionated without unnecessary hedging.
